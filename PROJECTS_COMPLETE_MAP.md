## NOTES ON DUPLICATE REGISTRATIONS

Several projects appear multiple times with different workspace IDs:

**Why?**
- When you moved projects from ~/Documents/VS Code Projects to ~/Projects/
- VS Code created new workspace IDs for the same folder
- Old entries weren't cleaned up, so multiple IDs coexist
- Each ID has its own session store

**Effect:**
- Opening via "Open Recent" or reopen may pick different IDs
- Chat History may show different subset of sessions
- All sessions still exist in state.vscdb; just indexed under different IDs

**Recommendation:**
- Choose one ID per project and consolidate
- OR keep as-is (sessions still recoverable, just fragmented)
- OR clean up stale workspace IDs from workspaceStorage
